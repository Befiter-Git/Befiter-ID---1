I need you to make specific changes to how phone numbers are handled in the identity service. Read this carefully and implement exactly what is described.

Change 1 — Phone column structure

The befiter_ids table currently has one phone column with a UNIQUE constraint.

Change it to:

Rename phone to current_phone — this is the active phone number used for all communication and lookup
Add a new column previous_phones as a text[] (text array) — this stores old phone numbers that this person previously used. Default is an empty array.
Remove the UNIQUE constraint from current_phone — the same phone number can exist on multiple records because phone numbers can be recycled by telecom operators
Change 2 — Create endpoint

Currently the create endpoint checks for duplicate phone and returns 409 if found.

Change it to:

Only check for duplicate current_phone match — do NOT return 409 if phone already exists. Allow it and create the new identity.
Still check for duplicate email and return 409 if email already exists — email remains unique and is enforced at database level
Change 3 — Lookup endpoint

Currently the lookup searches phone first, then email.

Keep it exactly the same BUT only search current_phone column — never search previous_phones.

Change 4 — Create endpoint duplicate handling (Case 2)

This is the most important change. When a new identity create is attempted:

Step 1: Check if email already exists in befiter_ids

If email EXISTS (same email, any phone):

Do NOT create a new record
This is the same person
Compare the phone in the request with current_phone on the existing record
If phone is different:
Move the existing current_phone into the previous_phones array
Set the new phone as current_phone
Return the existing identity record
Create an app_link for the calling app
Return 200 (not 201, since nothing new was created)
If email does NOT exist:

Create a new identity record normally
Return 201

Different	Same	Create new identity. Phone being same is acceptable.
Different	Different	Create new identity.

When lookup returns a found identity, include previous_phones in the response so calling apps can see the full phone history.
